Frequently Asked Questions

Security and Privacy

My Windows PC flags the application as a security concern. Is this application a security threat?

No, this happens because Windows does not trust unsigned executables and may flag them as potential malware. The warning appears because our executable lacks a valid certificate authority (CA), which requires a costly certification process that is not necessary for this software. You can manually trust the executable, as it is safe and not malware.

My antivirus software detected the application as a trojan. Is this true?

No, this is a false positive. You can manually mark our software as trusted, as we have no reason to distribute a trojan, nor would it be legal to do so. If you still have concerns, you may request a refund.

Are my Twitter credentials safe?

Yes, your Twitter credentials are highly secure because they are encrypted using a machine-specific and user-specific key. Our application features enterprise-level security as the encryption key is derived using PBKDF2 with SHA-256 and a 16-byte user-specific salt at each runtime, and this key encrypts your data using AES-128 in CBC mode with HMAC-SHA256 authentication, ensuring that only your user account and your device can decrypt the stored data. This makes any attempt to steal your credentials remotely impossible.

Do you collect any sensitive data from us?

No, we never collect any sensitive data from you, including your Gmail account or Twitter credentials. We use enterprise-level encryption to protect your sensitive data, which is then securely stored on your device. To learn how your data is used, please see our Privacy Policy. Note that while we do not store any sensitive data, we do store the email address of the PayPal account used for your purchase. This email is used for license activation and verification.

When I try to authenticate using Gmail OAuth, I get a warning that says Google hasn't verified this app. How do I grant access to the application?

Our application requires read-only access to Gmail, which triggers Google's security verification process. Unfortunately, Google mandates a CASA (Cloud Application Security Assessment) verification for applications requesting sensitive scopes, such as the Read-Only scope. CASA verification is a very costly security assessment that must be completed annually, and we currently do not have sufficient funds to cover the verification cost.

Because of this, our application cannot be verified and Google will always display a warning when you try to use it. To proceed, click on Advanced and select Go to Skeb Sniper (unsafe). If you want to learn how your data is processed, we encourage you to read our Privacy Policy. If you are not comfortable using our software, you may request a refund. Once the refund is processed, your usage privileges will be revoked.

Application Requirements

What are the requirements for running this application?
  • Ensure you have sufficient Skeb points for the request or that your bank does not require authentication. Otherwise, the program will fail to submit your request.
  • Make sure you are using your Gmail account to receive emails from Skeb. If not, update your email to your Gmail account in your Skeb account settings.

Can I run this application on my own PC?

Yes, you can run the application on your PC, but it is recommended to use a server instead. Running it on a personal computer is not ideal because most users cannot keep their PC running 24/7, and doing so is not good for your hardware.

Am I required to enter my Twitter credentials?

Yes, the program requires your credentials to log in. While they're not always needed, it's safest to have them available. When submitting a request, the program will first try to use cookies if you have already logged into your Firefox account. If that fails, it will fall back to your Twitter credentials.

I do not use Gmail to receive emails from Skeb. Can I not use Gmail?

Unfortunately, Skeb Sniper currently only works with Gmail. Each email API is different, and supporting multiple email APIs would require significant time and effort. Since this software is developed and maintained by a single developer, we currently lack the resources to support additional email APIs. We chose the Gmail API because it's the most popular choice. If you prefer not to use Gmail, this program will not function as intended.

Application Functionality

How does this program work?

Skeb Sniper is an email driven program, meaning that only when you receive an email from Skeb stating that the artist is open for requests, the program will send a request to the artist.

What does "Run in Background" in task manager do?

You use "Run in Background" when you genuinely want to send a request to the artist. Do not use it for testing because no fake email will be sent to you to trigger the automation tool. Automation will only be triggered by the email sent from Skeb. When you click on "Run in Background," the program starts a background process that runs continuously, allowing you to safely disconnect from your server. On Windows, it uses a Windows service, while on Linux, it runs as a daemon managed by supervisord, so you need admin/root privileges to run. The background process automatically shuts down when no tasks are running.

You advertised the application as "sending an instant request." How fast is "instant"?

According to the developer's tests on Windows and Linux servers, using the "Run in Background" feature (not "Test Run," which may take longer to distinguish scenarios), from the time the artist opens requests on Skeb to the time when the application successfully sends a request, it takes about 8 seconds. However, the response time may vary depending on factors such as server latency on Skeb and Google, computer performance. If no errors occur, your request should be sent well under 1 minute.

Does sending a request through this application guarantee that the artist will accept my commission?

No, this application is designed to automatically send a request to the artist, replacing the need for manual submission, so you don’t have to wake up at ungodly hours to send the request yourself. However, whether your request will be accepted depends on the artist and is beyond this program’s control.

Tutorial

How do I test the program?

You can test the program by clicking on "Test Run." The program will send you an automated email, which will trigger the automation tool to send a request to the artist. We recommend selecting an artist currently open for requests to confirm that the program can log into your account. If login fails, ensure that Two-Factor Authentication is disabled on Twitter. If you prefer not to send a request right now, set the amount to a value below the minimum requirement (e.g., 500). If test run fails, please contact the developer.

How do I confirm that the program is listening for my emails?

You can verify this by checking the log file named "GmailListener.log" in your working directory after running a task. If you see "Gmail authentication successful!", "Gmail Watch Started!", and "Listening for messages...", the program is actively listening for new emails.

I am running a task for an artist that is currently open in background, but the program does not send a request to the artist. Why is that?

Skeb Sniper is an email driven program, so if the artist is currently open, no email will be sent to you from Skeb, thus nothing will trigger the automation tool and no request will be sent. Only use "Run in Background" for artists that are currently closed and ensure that you have reserved email notification from Skeb.

How do I choose the right server to run the application?

Your choice of server depends on your technical expertise and budget:

  • Windows Version: Good for users with little to no technical experience in computers as Windows provides a user-friendly graphical user interface (GUI) that handles all tasks without command-line operations.
  • Linux Version: Recommended for users with technical expertise seeking a cost-effective solution. Linux servers are significantly cheaper than Windows servers, but a basic understanding of Linux systems is required to use them effectively.
What are the minimum server requirements?

Assuming no resource-intensive programs are running on your server, the minimum requirements for running the program are:

  • Linux: 2GB of RAM and a 10GB persistent disk.
  • Windows: 4GB of RAM and a 50GB persistent disk.

Windows requires more RAM and disk space because it includes a built-in graphical user interface (GUI).

Linux-Specific Questions

Why do you offer a Linux version of the software?

Our application is a long-running program that continuously monitors emails from Skeb, therefore it is better to run the application on a server than on your own PC. Most people are familiar with Windows, but due to high Microsoft licensing costs, a Linux server is a much more affordable alternative.

For example, on Google Cloud Platform as of March 18, 2025, running the program on Linux requires 2GB of RAM and a 10GB persistent disk, costing approximately $13 per month. In contrast, a Windows setup requires 4GB of RAM and a 50GB persistent disk, costing $63 per month. This means that using a Linux server reduces costs to about 20% of a Windows server.

Therefore, Linux is the preferred choice for server deployments due to its cost efficiency. However, if you are not familiar with Linux, we recommend using Windows, as Linux requires more technical knowledge to set up and manage.

Do you have a tutorial on running the program on a Linux server?

Currently, we do not provide a tutorial for running our software on a Linux server, as we expect Linux users to be familiar with basic commands and GUI forwarding. If you are unsure how to set up GUI forwarding, there are video tutorials available on YouTube that can guide you through the process. If you are unable to get GUI forwarding to work, consider using the Windows version instead.

How do I run the application on a Linux server?

Before running the application, make sure Docker is installed and running on your Linux server. Download the shell scripts setup.sh and run.sh from the Purchase tab.

To set up the container, run:

./setup.sh

Since the application requires GUI forwarding, the Docker image includes X11 forwarding to display the GUI on your local PC. If you are using a Windows PC, we recommend using PuTTY and Xming for X11 forwarding. Note that X11 forwarding can be slow, so expect some delay after running the program.

To launch the GUI, run:

./run.sh

You can use this command anytime you want to start the GUI.

Can I use setup.sh on any Linux system, including my Windows Subsystem for Linux (WSL) on my Windows PC?

Yes, setup.sh will work on any Linux system as long as your Linux system is properly configured with Docker installed, an X11 environment with DISPLAY set. The shell script loads the Docker image, copy the necessary files to your local directories, and runs the skeb-sniper-container. Note that WSL’s --network host option may not fully replicate native Linux networking behavior, but this will not prevent the script from executing successfully on a properly configured Linux system.

Can the program still launch a browser after disconnecting from my Linux server?

Yes. When you use SSH (e.g., with PuTTY), the application's GUI and Firefox use X11 forwarding. If SSH is not active, the GUI will be displayed using Xvfb. So regardless of the scenario, the application will always be able to launch a browser.

Why do I get the error "PuTTY X11 proxy: unable to connect to forwarded X server: Network error: Connection refused" after running setup.sh using PuTTY with X11 forwarding?

If you’ve successfully connected to your Linux server via SSH using PuTTY, make sure that:

  • X11 forwarding is enabled on the Linux server.
  • Your X server (e.g., Xming) is running on your Windows PC.
  • No firewall is blocking the X11 connection.

The shell script setup.sh is designed to automatically resolve any xauth mismatch by updating the container's xauth to match the host's.

Tips for How to Send A Successful Request

Amount

Recommended Amount: If the amount you entered is below the recommended amount, Skeb may prevent the request from being sent. If you are unsure about the recommended amount, you can set a multiplier in the application.

Text Validity
  • Language and Translation: Skeb uses automated translation to bridge language barriers between clients and creators. To ensure clarity, compose your request in a straightforward language and avoid idiomatic expressions or complex sentences that may not translate accurately.
  • Prohibited Terms: Certain words or phrases might be flagged by Skeb's system. For example, mentioning "日本語" ("Japanese") in your request may result in rejection, as Skeb does not accept machine translations. Note that Skeb uses DeepL for translation.
  • Clarity and Specificity: Given that direct communication with creators after sending the request is prohibited by Skeb, it's essential to include all necessary details in your request. Be specific about what you want in the artwork to ensure the final result matches your expectations.
  • Character Limit: Know the maximum character limit set by the artist (usually 1000). Exceeding this limit may result in text truncation when you use the application to send requests.
Reference URL
  • Accessibility: If your request includes a reference URL, ensure that the link is publicly accessible. For platforms like Google Drive, set the sharing settings to "Anyone with the link."
  • Including Reference URL in your Text: The reference URL input may not appear on Skeb if your message is too short. Skeb only provides this option when longer text is entered. To ensure your reference URL is included, always include your reference URL in your text.
NSFW Content
  • NSFW Setting: If your request involves NSFW content, make sure to set the NSFW toggle to "On" when entering your request using the application.
  • Compliance with Guidelines: Even with the NSFW toggle enabled, certain extreme or prohibited content may still violate Skeb's Terms of Service. Review Skeb's guidelines before submitting.
Payment
  • Points: Ensure you have enough Skeb points before submitting a request. The application cannot send requests without sufficient points unless your bank does not require authentication. Using a credit card may trigger 3D authentication, which automation cannot bypass. You can purchase points on Skeb, but be aware that they have an expiration date and may expire if not used.
  • Credit Cards: We do not recommend using credit cards, as most banks in America require authentication. Because of this, our automation tool cannot send a request if authentication is required. However, if you are certain that your bank does not require authentication, you can test it on Skeb by attempting to send a request using your credit card. If you are not redirected to 3D authentication, you can use your credit card without needing to purchase points.
