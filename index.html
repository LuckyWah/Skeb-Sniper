Frequently Asked Questions

My Windows PC flags the application as a security concern. Is this application a security threat?

No, this happens because Windows does not trust unsigned executables and may flag them as potential malware. The warning appears because my executable lacks a valid certificate authority (CA), which requires a costly certification process that is not necessary for this software. You can manually trust the executable, as it is safe and not malware.

What are the requirements for running this application?
  • Ensure you have sufficient Skeb points for the request or that your bank does not require authentication. Otherwise, the program will fail to submit your request.
  • Make sure you are using your Gmail account to receive emails from Skeb. If not, update your email to your Gmail account in your Skeb account settings.
  • Confirm that you have enabled notifications for the artist. If not, visit the artist's Skeb page and click "Notify on seeking started."
  • Test the program and successfully activate the Gmail listener before running the task in the background.
  • When running the task in the background, ensure the Gmail listener is activated successfully.
Are my Twitter credentials safe?

Yes, your Twitter credentials are safe because they are encrypted using a machine-specific and user-specific key. The encryption key is derived using PBKDF2 with SHA-256 and a 16-byte user-specific salt, and this key encrypts your data using AES-128 in CBC mode with HMAC-SHA256 authentication, ensuring that only your user account and your device can decrypt the stored data.

Do you collect any data from us?

No, the developer never collects any data from you, including your Gmail account or Twitter credentials. To ensure your privacy and security, all sensitive data is first encrypted and then securely stored locally on your device in compliance with our no-data-collection policy.

Do I have to input my Twitter credentials?

Yes, the program requires your credentials to log in. However, it will first attempt to use cookies. If that fails, it will fall back to using your password.

How do I test the program?

You can test the program by clicking on "Test Run." The program will send you an automated email, which will trigger the automation tool to send a request to the artist. We recommend selecting an artist currently open for requests to confirm that the program can log into your account. If login fails, ensure that Two-Factor Authentication is disabled on Twitter. If you prefer not to send a request right now, set the amount to a value below the minimum requirement (e.g., 500). If test run fails, please contact the developer.

What does "Run in Background" in task manager do?

When you click on "Run in Background," the program starts a background process that runs continuously, allowing you to safely disconnect from your server. On Windows, it uses a Windows service, while on Linux, it runs as a daemon managed by systemd, so you need admin/root privileges to run. The background process automatically shuts down when no tasks are running.
