Trust Center

Our software is designed with user privacy and security in mind, and we are not just paying lip service. If you are privacy-conscious, you do not have to take our words blindly. You are the judge, and you decide whether Skeb Sniper is safe. You can verify our claims regarding privacy protection by following these simple tests.

1. Your Twitter Credentials Are Never Transferred and Only Stored Locally

You can verify that Skeb Sniper never transmits your Twitter credentials and encrypts them locally:

  1. Launch Skeb Sniper. The software will perform an online license check. Note that the program checks the license every hour, so complete this test within one hour.
  2. Enter a random Twitter username and password.
  3. Block all internet access for Skeb Sniper using your firewall or network monitor.
  4. Create a task using the random Twitter credentials.
  5. Locate the encrypted credentials in a text file named "user info" inside Skeb Sniper’s installation directory.
  6. Open the encrypted file with a text editor. You will see unreadable encrypted data. The program will not crash or throw errors, because there is no code attempting to transmit your credentials.
  7. Monitor your network activity. You will see no outgoing connections during password entry.

This confirms that password is handled locally without transmitting any sensitive information.

2. We Use User-Specific and Machine-Specific Encryption

You can verify that even if the encrypted credentials file is stolen, it cannot be decrypted using another user account on the same machine or a different machine:

  1. Copy all files in your working directory, including the encrypted user information file, to another user account on the same machine or to a different machine.
  2. Launch Skeb Sniper. The software will automatically attempt to decrypt the file.
  3. Observe that no user information is shown in Skeb Sniper. This indicates that decryption has failed, and the program has deleted any files it cannot decrypt because the encryption key is different.

This confirms that Skeb Sniper’s encryption is tied to both your specific device and user account, preventing unauthorized decryption even if the file is stolen.

3. Our Software Processes Emails Locally Without Transmission

You can verify that Skeb Sniper processes email notifications locally and never transmits your email data to our server:

  1. Launch Skeb Sniper normally. This step requires contacting our server once to check your license. Complete this within one hour.
  2. Block all connections to our server kasorashibainu.com using your firewall.
  3. Send a test email to your own Gmail account.
  4. Observe that Skeb Sniper still detects the new email correctly in the log, even with our server blocked.

This confirms that Skeb Sniper does not transmit your Gmail emails or metadata externally. Email notifications are processed entirely on your device.

By performing these tests, you can be confident that you have absolute control over your own credentials and that your emails are never read by us.
